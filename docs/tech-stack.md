## 6. 選定理由の詳細

### なぜ React か

- 世界的に最もシェアが高く、日本国内でも求人数が最多
- コンポーネント単位でUIを管理でき、カードやカラムの実装に向いている
- 学習リソース（公式ドキュメント・Qiita・YouTube）が豊富

### なぜ TypeScript か

- JavaScriptに「型」という仕組みを追加した言語
- 変数に入れられる値の種類（文字列・数値など）をあらかじめ決めることで、実行前にミスを発見できる
- バックエンドがJavaで型を使う設計のため、フロントも型を合わせると一貫性が生まれる

### なぜ Java + Spring Boot か

- Javaは国内の業務システムで圧倒的に使われており、求人数も多い
- Spring Boot はJavaでAPIサーバーを作る際の業界標準フレームワーク
- 型安全・オブジェクト指向の設計がしっかりしており、大規模開発にも耐える

### なぜ Spring Data JPA か

- SQLを直接書かずに、Javaのコードだけでデータベースを操作できる
- Spring Boot との統合が最もシンプルで、追加設定が少ない

### なぜ PostgreSQL か

- 業界標準のリレーショナルDB。無料で使えて実績が豊富
- Spring Data JPAとの相性が良い

### なぜ Tailwind CSS + shadcn/ui か

- Tailwind CSS はクラス名を並べるだけでスタイルを当てられるため、CSSファイルを別途書く必要がない
- shadcn/ui はコピーして使うUIコンポーネント集で、デザインを自由にカスタマイズできる
